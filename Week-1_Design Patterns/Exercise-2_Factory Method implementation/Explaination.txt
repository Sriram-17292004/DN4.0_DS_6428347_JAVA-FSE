This implementation uses a **Factory Method Pattern** with a single `DocumentFactory` class that creates different document types (`Word`, `PDF`, `Excel`) based on a string input. Instead of separate factory classes for each document type, the `createDocument()` method takes a string argument (e.g., `"word"`, `"pdf"`) and returns the corresponding `Document` object. The factory also includes error handling for invalid inputs and an optional helper method to streamline document processing. We can verify the same using DocumentFactoryTest class.
